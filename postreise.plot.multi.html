postreise.plot.multi.plot_bar.plot_bar(interconnect, time, scenario_ids=None, scenario_names=None, custom_data=None, resource_types=None)[source]¶

Plot any number of scenarios as bar charts with two columns per scenario - defaults to generation and capacity.

Parameters

  • interconnect (str) – either ‘Western’ or ‘Texas’.

  • time (tuple) – time related parameters. 1st element is the starting date. 2nd element is the ending date (left out). 3rd element is the timezone, only ‘utc’, ‘US/Pacific’ and ‘local’ are possible. 4th element is the frequency, which can be ‘H’ (hour), ‘D’ (day), ‘W’ (week) or ‘auto’.

  • scenario_ids (list) – list of scenario ids, defaults to None.

  • scenario_names (list) – list of scenario names of same len as scenario ids, defaults to None

  • custom_data (dict) – hand-generated data, defaults to None

  • resource_types (list) – list of resource types to show, defaults to None

Note

If you want to plot scenario data and custom data together, custom data MUST be in TWh for generation and GW for capacity. We may add a feature to check for and convert to equal units but it’s not currently a priority

postreise.plot.multi.plot_helpers.handle_plot_inputs(interconnect, time, scenario_ids, scenario_names, custom_data)[source]¶

Check input validity for plotting code, fetches data if necessary.

Parameters

  • interconnect (str) – either ‘Western’ or ‘Texas’

  • time (tuple) – time related parameters. 1st element is the starting date. 2nd element is the ending date (left out). 3rd element is the timezone, only ‘utc’, ‘US/Pacific’ and ‘local’ are possible. 4th element is the frequency, which can be ‘H’ (hour), ‘D’ (day), ‘W’ (week) or ‘auto’.

  • scenario_ids (list) – list of scenario ids.

  • scenario_names (list) – list of scenario names of same len as scenario ids.

  • custom_data (dict) – hand-generated data.

Raises

ValueError – if interconnect is not Western or Texas, the number of scenario names is not equal to number of scenario ids, the number of scenario ids and/or custom data is less than two.

Returns

(list) – list of all zones in interconnect and formatted graph data

postreise.plot.multi.plot_shortfall.plot_shortfall(interconnect, time, scenario_ids=None, scenario_names=None, custom_data=None, is_match_CA=False, has_collaborative_scenarios=None, baselines=None, targets=None, demand=None)[source]¶

Plot a stacked bar chart of generation shortfall for any number of scenarios.

Parameters

  • interconnect (str) – currently only ‘Western’ works. We include this param to keep a consistent API with all plotting functions.

  • time (tuple) – time related parameters. 1st element is the starting date. 2nd element is the ending date (left out). 3rd element is the timezone, only ‘utc’, ‘US/Pacific’ and ‘local’ are possible. 4th element is the frequency, which can be ‘H’ (hour), ‘D’ (day), ‘W’ (week) or ‘auto’.

  • scenario_ids (list) – list of scenario ids.

  • scenario_names (list) – list of scenario names of same len as scenario ids.

  • custom_data (dict) – hand-generated data, defaults to None.

  • is_match_CA (bool) – calculate shortfall using special rules that apply when all zones match California goals, defaults to False

  • has_collaborative_scenarios (list) – list of scenario ids where all zones collaborate to meet goals. Affects results for interconnect, defaults to None.

  • baselines (dict) – baseline renewables generation for each zone, defaults to None.

  • targets (dict) – target renewables renewable generation for each zone, defaults to None.

  • demand (dict) – total demand for each zone, defaults to None

Note

If you want to plot scenario data and custom data together, custom data MUST be in TWh for generation and GW for capacity. We may add a feature to check for and convert to equal units but it’s not currently a priority.
